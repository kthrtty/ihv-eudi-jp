# /retro [焦点]

対象はプロダクトではなく**進め方とハーネス自体**。人間への確認なしで改善を1周実装する。

## 手順
1. **素材収集**(自分で実行): `git log --oneline -20`・`memory/L1-working/*`・`memory/L2-episodic/` の直近ファイル・`memory/L3-semantic/lessons.md`・直近の会話で起きた失敗/手戻り/ユーザー指摘を箇条書きに(300字ブリーフ)
2. **並列批評**(サブエージェント3体を1メッセージで並列起動、ブリーフを渡す):
   - qa: 「この期間の手戻り・見逃しはプロセスのどこで防げたか」失敗シナリオ列挙
   - consultant: 「このハーネス/進め方の最も痛い欠陥」を悪魔の代弁者として
   - engineer: 「ハーネス(scripts/skills/hooks/memory規約)の技術的改善」を具体パッチ案で
3. **相互反証**(1ラウンド): 各批評の要旨を他の2体へ SendMessage で送り返し、反論・補強を回収
4. **裁定**: pm サブエージェントに全材料を渡し、採用する改善を3件以内に絞らせる(却下理由も)
5. **即実装**: 採用改善のうちハーネスファイル(scripts/ skills/ agents/ harness/ memory規約)の修正は**このまま実装・検証**する。プロダクトコードに触るものは L1 の次アクションに積む。
   **安全弁(協議レビュー 2026-07-07 裁定・変更禁止)**:
   - 前提: ハーネスファイルが git 管理済みかつクリーンであること(巻き戻し地点の確保)。未コミット変更があれば先にコミット
   - 変更後: `bash -n` 構文チェック + 各フックスクリプトの空実行スモーク(exit 0 確認)を必須とする
   - **人間ゲート(即質問禁止の唯一の例外)**: retro/SKILL.md 自身・.claude/settings.json の hooks 節・agents の権限記述の変更は、実装せず判断依頼フォーマットで人間に提示する(自己書換の遮断器)
   - net-zero 規則: 追加1件につき、既存の規則/仕組みの削除・簡素化を1件対にする(単調肥大の抑止)
   - 各改善に「効果の測定条件」を付し、次回 retro で効果不明なら revert を既定とする
6. **記録**: 議論→L2、教訓→L3 lessons.md(+INDEX)、L1 上書き
7. **上流還元**(唯一人間が関与する点): 複数プロジェクトで通用する改善は
   `patches/harness-<date>-<slug>.patch`(リポジトリ内・gitignore 済み)としてテンプレ正本($AGENT_VAULT/00-global/templates/project-harness)への差分を生成し、
   「適用コマンド+効果1行」を最終報告に含める(適用判断は人間。グローバル規約の一方向昇格)

規律: 改善は測定可能に(「〜を徹底する」ではなく仕組み/ファイル/フックに落ちる形)。3件超は却下。自己批判のないレトロは失敗とみなしやり直す。
