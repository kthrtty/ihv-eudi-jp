# /loop <トピック>

harness/loop.md のプロトコルで議論ループを実行せよ。

1. **Framing**: トピックを1文で定義。L0/L1から共有ブリーフ(300字以内)を作成。関与ロールを3〜5名選定
2. **Propose**: 起案担当のサブエージェント(既定: pdm、技術系: engineer、体験系: uiux)にブリーフを渡して起案させる
3. **Review**: 選定ロールのサブエージェントを**並列**起動し、各自の観点レビューを収集
4. **Challenge**: consultant サブエージェントに全見解を渡し最強の反論を構築させる。qa に失敗シナリオを列挙させる
5. **Decide**: pm サブエージェントに全材料を渡し裁定させる
6. **Record**: 各エージェントの「L2記録用サマリ」を memory/L2-episodic/$(date +%F)_<slug>.md へ追記。決定は decisions/ADR-XXXX.md + L0索引へ。未解決は open-questions.md へ

規律: ラウンド上限3。人間への確認が必要なら loop.md の判断依頼フォーマットで提示し、即質問はしない。
