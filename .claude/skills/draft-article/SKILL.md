# 記事ドラフト作成

Digital Identity領域のニュースを元に、レビュー可能な記事ドラフトを `editorial/drafts/` に生成する。

## 手順

1. **編集方針の確認**: `editorial/topics.md` を読み、指定されたURL/トピックが対象トピック(Tier 1-3)に該当するか確認する。該当しない場合はその旨を伝え、書くべきでない理由を添えて中止する。

2. **情報収集**:
   - 指定URLを WebFetch で読む。一次情報(仕様書・公式発表)でない場合は、記事中のリンクや WebSearch で一次情報を特定して必ず読む
   - 関連する標準・過去の動きを WebSearch で補足(topics.mdの文脈で)
   - 収集した事実は「どのURLに書いてあったか」を記録しながら進める

3. **ドラフト生成**: `editorial/templates/article-draft.md` の構成に従い、`editorial/drafts/YYYY-MM-DD_<slug>.md` を作成する。
   - **事実と意見を分離**: 「何が起きたか」は帰属表現(〜と発表された)で事実のみ。「分析」「Musubiaの視点」に解釈を書く
   - **検証メモを必ず埋める**: 本文の主要な主張それぞれに根拠URLを対応付ける(レビュアーが裏取りできる形)
   - **引用は短く**: 直接引用は和文数行以内+出典明記。長文転載・全訳はしない
   - **実務への示唆**: 読者(企業のID/セキュリティ担当者)が明日から何をすべきかを1段落必ず書く
   - 分量はtopics.mdの記事タイプ別目安に従う

4. **報告**: 生成したファイルパスと、TL;DR、レビュアーに特に見てほしい箇所(自信のない主張・裏取りが弱い箇所)を正直に報告する。

## 禁止事項
- ドラフトを本人の承認なしに `musubia-hp/news/` へ公開しない(公開は別途明示指示があったときのみ)
- 一次情報を読まずに二次報道だけで書かない
- 出典のない数値・固有の主張を書かない
